# Arcana License Lifecycle

## Purchase flow

```
User visits arcana.otnelhq.com
  → clicks "Pro $19/mo"
  → arcana-site POST /api/create-sub
  → proxy POST /v1/pay/create-sub
  → PayPal creates subscription → returns approval_url
  → user redirected to PayPal → completes payment
  → PayPal fires BILLING.SUBSCRIPTION.ACTIVATED webhook
  → proxy generates ARCANA-PRO-<base64> key
  → stores license in KV with expiry
  → emails key to user
  → user runs: arcana license activate <key>
```

### Expiry

| Plan | Price | Expiry | Renewal |
|------|-------|--------|---------|
| Pro Monthly | $19/mo | 30 days | PAYMENT.SALE.COMPLETED webhook → +30d |
| Pro Yearly | $190/yr | 365 days | PAYMENT.SALE.COMPLETED webhook → +365d |

### Renewal

PayPal auto-bills. On success, the `PAYMENT.SALE.COMPLETED` webhook extends
`sub.expiresAt` by 30 or 365 days and adds another 500 proxy credits. On
failure, `PAYMENT.SALE.DENIED` sets `sub.renewalAtRisk = true` (no immediate
revocation). After 3 failed attempts, PayPal suspends the subscription;
`BILLING.SUBSCRIPTION.SUSPENDED` sets `sub.status = "suspended"`. The license
key remains valid until its natural `expiresAt`.

### Cancellation

User cancels via PayPal → `BILLING.SUBSCRIPTION.CANCELLED` webhook → proxy
sets `sub.status = "cancelled"`. The license key is NOT immediately revoked.
It continues working until its `expiresAt`. No refund is processed by the
proxy; disputes are handled in PayPal.

## License types

### Subscription keys (automated)
- Generated by the proxy webhook handler
- Format: `ARCANA-PRO-<32-byte base64url>`
- Expiry: 30d (monthly) or 365d (yearly)
- Stored in proxy KV under `license:<key>` and `sub:<subId>`
- Email-indexed under `email_account:<email>`

### Admin-created keys (manual)
- Created via `POST /api/license/create` on the license server
- Requires `ARCANA_ADMIN_KEY` bearer token
- Default expiry: 365 days from creation
- Max expiry: 3650 days (10 years) — `MAX_EXPIRES_IN_DAYS`
- Format: `ARCANA-<TIER>-<timestamp36>-<16-byte hex>`

### Seed keys (provisioned)
- Loaded from `ARCANA_SEED_KEYS` env var (JSON)
- Survive redeploys — stored in env, not KV
- Default expiry: 365 days from worker start if no explicit `expiresAt`
- Used for dev/testing/enterprise provisioning

## Validation flow

```
arcana doctor / any command
  → client POST /api/license/validate (license server)
  → resolveLicense(key, kv, seedKeys)
  → check expiresAt > Date.now()
  → check machine is activated
  → return { valid, tier, features, tools, limits, expiresAt, seatsUsed }
```

The proxy also validates: on every authenticated request, `getUser()` checks
the license key against proxy KV, Workers Cache, or falls back to calling
the license server's validate endpoint.

## Expiry handling

| State | Behavior |
|-------|----------|
| `Date.now() > expiresAt` | License invalid. Validate returns 402. |
| `sub.renewalAtRisk` | License still valid. Warning surfaced in CLI. |
| `sub.status = "suspended"` | License still valid until expiry. No new machines. |
| `sub.status = "cancelled"` | License valid until expiry. No renewal. |
| `sub.status = "expired"` | License invalid. Key is dead. |

## Credits

Pro subscriptions include 500 credits (~$5) on activation and each renewal.
Credits are used for LLM API costs via the proxy. Non-enterprise users are
pre-charged the estimated max cost; the difference is refunded after the
call completes. Credit balance lives in proxy KV under `balance:<userId>`.

## Architecture

```
arcana-site (Pages)          proxy (Worker)              license-server (Worker)
      │                           │                              │
      │  /api/create-sub          │                              │
      │──────────────────────────>│                              │
      │                           │  PayPal API                  │
      │                           │─────────────────────────────>│
      │                           │                              │
      │                           │  PayPal webhook              │
      │                           │  → generate key              │
      │                           │  → store in proxy KV         │
      │                           │                              │
      │                           │                              │
      │                    [arcana CLI]                           │
      │                           │  /api/license/validate       │
      │                           │─────────────────────────────>│
      │                           │  { valid, tier, ... }        │
      │                           │<─────────────────────────────│
```
